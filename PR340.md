Extends the `config bootstrap` command to support importing data from Terra Workspaces using the fs.anvilfs PyFilesystem2 plugin. This enhancement enables seamless bulk configuration of Galaxy instances with data from Terra workspace buckets.

The implementation adds a new `terra_workspaces` section to bootstrap YAML configurations, supporting file pattern matching, custom history assignment, and automatic datatype detection. The fs.anvilfs library provides a clean filesystem abstraction over Terra workspaces, handling authentication and workspace enumeration while leveraging the enhanced dataset configuration features added in #339.

Integration follows the existing bootstrap command patterns, utilizing Galaxy's URL-based import mechanisms and maintaining full backward compatibility with existing configuration files.
